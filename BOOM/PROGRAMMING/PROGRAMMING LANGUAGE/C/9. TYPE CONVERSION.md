
Most of the primitive types are allowed to convert to each other without any explicit type casting.
Some of these conversions cause loss of information.

float x =10.5;
int y=x;      // y=10
bool z=y;   // z=1

If we have an expression of multiple types, below rules are followed

bool -> char -> short int -> int -> unsigned int -> long -> long long -> float -> double -> long double.

```c
#include<stdio.h>
int main()
{
   int x=10;
   float y=10.5;
   float z=x+y;
   printf("%f",z);
   return 0;
}
```

Output:
20.5


```c
#include<stdio.h>
int main()
{
    char a = 'B';
    double b = 10.5;
    double c = a+b;
    printf("%g",c);
    return 0;
}
```

Output:
76.5


## Explicit Conversion

```c
#include<stdio.h>
int main()
{
    int x=15,y=2;
    double z=x/y;
    printf("%g",z);
    return 0;
}
```

Output:
7

```c
#include<stdio.h>
int main()
{
    int x=15,y=2;
    double z=(double) x/y;
    printf("%g",z);
    return 0;
}
```

Output:
7.5
