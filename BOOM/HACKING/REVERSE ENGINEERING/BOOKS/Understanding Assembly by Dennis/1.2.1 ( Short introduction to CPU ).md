The **Central Processing Unit (CPU)** is the primary device that executes the **machine code** making up a program. Key concepts in this section include:

- **Glossary of Terms**:
    - **Instruction**: A primitive command for the CPU, such as moving data between registers or performing arithmetic.
    - **Machine Code**: The raw code processed directly by the CPU, usually encoded as a series of bytes.
    - **Assembly Language**: A human-readable mnemonic code (often including macros) designed to make programming at the low level easier than writing machine code directly.
    - **Compiler**: The software that translates high-level programming languages (like C++ or Java) into assembly or machine code.
- **CPU Registers**: These are fixed sets of "general-purpose registers" (GPR) that act as **untyped temporary variables**. Common architectures have varying counts: x86 has approximately 8, while x86-64 and ARM have approximately 16.
- **Instruction Set Architectures (ISA)**: Different CPUs use different ISAs.
    - **x86**: Known for variable-length instructions and maintaining legacy support for 16-bit instructions.
    - **ARM**: A **RISC** (Reduced Instruction Set Computing) architecture. It features several modes: "ARM mode" (4-byte instructions), "Thumb mode" (2-byte instructions for space efficiency), and "Thumb-2" (a mix of 2-byte and 4-byte instructions). **ARM64** uses 4-byte instructions exclusively.
    - **Other RISC ISAs**: This includes MIPS, PowerPC, and Alpha AXP, which traditionally use fixed-length 32-bit instructions.
