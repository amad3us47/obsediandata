
**The method** described in section 1.1 is a technique for learning and understanding how high-level code relates to low-level machine instructions. It involves a simple, repeatable workflow:

1. **Write small snippets of code**: Typically in C or C++, starting with very basic functions.
2. **Compile the code**: Use a compiler to translate the high-level code into machine instructions.
3. **Examine the assembly output**: Study the resulting assembly language to see exactly how the compiler translated the source code.

Purpose and Benefits

The primary goal of this method is to **imprint the relationship between C/C++ code and compiler output** onto the mind. By doing this many times, a reverse engineer develops the ability to look at high-level code and instantly imagine its underlying assembly structure, or conversely, look at assembly and understand the original C logic.

The author notes that this method is not just for beginners; they still use it when encountering obscure code they do not immediately understand.

Recommended Tools and Exercises

- **Compiler Explorer (godbolt.org)**: The source recommends this website as an excellent tool for this method, as it allows you to compare output from various compilers without installing them locally.
- **Manual Optimization**: A more advanced version of this method involves taking the assembly generated by the compiler and manually rewriting it to be as short as possible. While modern compilers are too efficient for this to be practical in real-world scenarios, it is a highly effective exercise for gaining a deeper mastery of assembly language
